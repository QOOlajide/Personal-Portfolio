# Experience / Skills Section — UI Contract (match Screenshot #2 format)

Goal:
- Experience (or Skills) section should match the second screenshot layout:
  - Big section headings with icon
  - Clean grid of pill items

Section structure:
- Section Title Row:
  - Left: square icon badge (rounded, subtle gradient allowed)
  - Right: Title text “Framework / Tools” with premium emphasis (can use subtle gradient text)
- Pill grid below title

Recommended subsections (V1):
1) Framework / Tools
2) Languages
(Optional V2: Cloud/Infra, AI/LLM tools)

Pill grid behavior:
- Desktop: 3 columns
- Mobile: 1–2 columns depending on width
- Pills are uniform height, rounded, subtle border or muted background

Pill style:
- Light surface on dark background (still muted)
- Minimal shadow
- No loud outlines
- Text centered, medium weight

What goes in the section:
- Framework/Tools: Next.js, React, TypeScript, Tailwind, FastAPI, Git, Docker, etc.
- Languages: Python, JavaScript, TypeScript, Java, SQL, HTML/CSS, Bash

Important:
- Keep this section scannable:
  - Do NOT dump 25 tools
  - Prefer 8–12 per subsection in V1
- The format should feel like “clean capability inventory,” not a badge collection
